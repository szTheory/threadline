# Plan 31-01: `Threadline.ChangeDiff` — core projection API

**Phase:** 31 — Field-level change presentation  
**Plan:** 31-01 of 02  
**Status:** Ready to execute

---

## Threat model

| ID | Threat | Severity | Mitigation in this plan |
|----|--------|----------|-------------------------|
| T-31-01 | **Information disclosure:** diff implies cleartext or prior values when capture did not store them | high | Implement CONTEXT D-3 exactly: `before_values` row signal; omit `before` when `"none"`; use `prior_state: "omitted"` when sparse key missing; never synthesize OLD scalars. |
| T-31-02 | **Integrity / deception:** stable bytes or field order leak integrator assumptions to downstream caches | medium | Lexicographic `field_changes` by `"name"`; sort any auxiliary field-keyed maps at construction; document `schema_version: 1`. |

---

## Objective

Add **`Threadline.ChangeDiff`** with a documented public entrypoint that returns a **deterministic, Jason-encodable** map for a given **`%Threadline.Capture.AuditChange{}`**, covering INSERT / UPDATE / DELETE and **`changed_from` nil vs map** per **`31-CONTEXT.md`** D-2–D-6, without new capture SQL or DB access.

---

## must_haves

- [ ] `lib/threadline/change_diff.ex` exists with `@moduledoc` covering: authority (pass-through only), INSERT/UPDATE/DELETE matrix, `before_values`, `prior_state`, `except_columns` / mask placeholders, `schema_version`, relationship to **`Threadline.Export`**.
- [ ] Primary shape: string keys, `"schema_version" => 1`, `"before_values"` ∈ `["none", "sparse"]`, `"field_changes"` as lexicographically sorted list of objects with at least `"name"` and documented `"after"` / prior epistemics.
- [ ] `format: :export_compat` option returns string-key map with at least `"op"`, `"data_after"`, `"changed_fields"`, `"changed_from"` plus identifiers **`"id"`, `"transaction_id"`, `"table_schema"`, `"table_name"`, `"captured_at"`, `"table_pk"`** aligned with **`Threadline.Export`** `change_map/1` **base** keys (same string coercion patterns: `id` and `transaction_id` as strings, `changed_fields` default `[]`, `changed_from` default `%{}` when nil). **Omit** `"action"` and `"transaction"` nested objects in Phase 31 — document that gap vs full export rows in `@moduledoc`.
- [ ] UPDATE iterates **only** `changed_fields` list (nil-safe); does not union-map keys from `data_after`.
- [ ] Options: `expand_insert_fields: true | false` (default `false`) per CONTEXT D-4.

## Task 31-01-01 — Module skeleton + primary wire format

<read_first>
- `.planning/phases/31-field-level-change-presentation/31-CONTEXT.md` (full)
- `lib/threadline/capture/audit_change.ex`
- `lib/threadline/export.ex` (`change_map/1` private function body for key naming parity)
- `lib/threadline/capture/trigger_sql.ex` (moduledoc through `changed_from` / `except_columns` summary)
</read_first>

<action>
1. Create **`lib/threadline/change_diff.ex`** with `defmodule Threadline.ChangeDiff`.
2. Implement **`@spec from_audit_change(Threadline.Capture.AuditChange.t(), keyword()) :: map()`** and **`def from_audit_change(%Threadline.Capture.AuditChange{} = ch, opts \\ [])`**.
3. Default `opts` format is **primary** (not `:export_compat`): build map with string keys including `"schema_version" => 1`, `"before_values"` set to `"none"` when `ch.changed_from` is `nil`, `"sparse"` when `ch.changed_from` is a map (including `{}`).
4. **UPDATE** branch (`ch.op == "UPDATE"`): build `"field_changes"` from `ch.changed_fields || []` only; each element `%"name" => col, "after" => ...` using `Map.get(ch.data_after || %{}, key)` for after-values (encode atoms like `:mask` as JSON would via existing patterns — use the same value pass-through as `change_map` / Jason for maps); apply D-3 prior epistemics; sort by `"name"` lexicographically.
5. **INSERT** branch: default `"field_changes" => []` (or omit — pick one and document); when `Keyword.get(opts, :expand_insert_fields, false) == true`, derive entries from `Map.keys(ch.data_after || %{})` sorted, each `"kind" => "set"` or documented equivalent per CONTEXT.
6. **DELETE** branch: document row removal shape — include `op`, identifiers, `"data_after" => nil` if using primary format; **no** synthetic field removals.
7. **Clauses:** function head validates `op` is one of `"INSERT"`,`"UPDATE"`,`"DELETE"`; unknown `op` raises `ArgumentError` with message containing **`unsupported op`**.
</action>
