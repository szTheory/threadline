## Tasks

### Task 1: Confirm canonical `origin` and drift

**What to do**

1. Show `git remote -v` and `git remote get-url origin`.
2. Fetch and compare SHAs: `git fetch origin main` and record `git rev-parse main` and `git rev-parse origin/main`.

**Read first**

- `.planning/REQUIREMENTS.md` (REPO-01, REPO-03)
- `mix.exs` ( `@source_url` for canonical URL string )

**Acceptance criteria**

- `git remote get-url origin` output contains the same GitHub repository path as `mix.exs` `@source_url` (e.g. `szTheory/threadline`).
- After `git fetch origin main`: either **(A)** `git rev-parse main` and `git rev-parse origin/main` differ ‚Äî record ‚Äúahead by N commits‚Äù using `git rev-list --left-right --count origin/main...main` ‚Äî or **(B)** they already match ‚Äî record ‚Äúalready in sync‚Äù with the single SHA printed in `08-01-SUMMARY.md`.

**Done condition:** Drift state (A or B) is recorded in `08-01-SUMMARY.md`.

---

### Task 2: Push `main` to `origin`

**What to do**

1. Run `git push origin main`.
2. If push fails, capture stderr in SUMMARY and stop ‚Äî do not mark REPO-03 complete.

**Read first**

- `.github/workflows/ci.yml` (confirm `push` ‚Üí `branches: [main]`)

**Acceptance criteria**

- After successful push: `git fetch origin main` then `[ "$(git rev-parse main)" = "$(git rev-parse origin/main)" ]` exits 0.

**Done condition:** Local `main` and `origin/main` refer to the same commit.

---

### Task 3 (optional): Local CI health before / after push

**What to do**

1. If Postgres is available, run `MIX_ENV=test mix ci.all` once before push to avoid pushing a broken HEAD (optional but recommended).

**Read first**

- `mix.exs` (`ci.all` alias)

**Acceptance criteria**

- If command was run: exit code 0 recorded in SUMMARY.
- If skipped: SUMMARY states ‚Äúskipped‚Äù with reason.

**Done condition:** SUMMARY records the choice.
