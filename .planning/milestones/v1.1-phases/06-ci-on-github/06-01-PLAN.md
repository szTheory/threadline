## Threat model

| ID | Threat | Severity | Mitigation in this plan |
|----|--------|----------|-------------------------|
| T-06-01-01 | Renamed or missing `jobs:` keys → docs/`act`/branch rules drift | high | Greps require literal `verify-format`, `verify-credo`, `verify-test` as YAML keys under `jobs:`. |
| T-06-01-02 | `main` no longer triggers CI → false confidence on integration branch | high | Greps require `branches: [main]` under both `push:` and `pull_request:`. |
| T-06-01-03 | Local `mix ci.all` passes but CI fails on warnings-as-errors | medium | Extend `ci.all` to run `compile --warnings-as-errors` before `verify.test` in the same env as today’s alias chain. |

---

## Goal

Lock **CI-01** (workflow on `main` with stable job keys and triggers) and remove the **local vs `verify-test` drift** called out in `06-RESEARCH.md` (warnings-as-errors only on GitHub today). After this plan, a contributor running `mix ci.all` gets the same compile strictness ordering as the `verify-test` job, so **CI-02** failures are reproducible locally before push.

## Tasks

### Task 1: CI-01 — Job keys and `main` triggers

**What to do**

1. Read `.github/workflows/ci.yml` and confirm three top-level job keys and `on:` branches match the v1.1 contract.

**Read first**

- `.github/workflows/ci.yml`
- `.planning/phases/06-ci-on-github/06-CONTEXT.md` (D-07, D-02)

**Acceptance criteria**

- `grep -qE '^  verify-format:' .github/workflows/ci.yml` exits 0.
- `grep -qE '^  verify-credo:' .github/workflows/ci.yml` exits 0.
- `grep -qE '^  verify-test:' .github/workflows/ci.yml` exits 0.
- `grep -A2 '^  push:' .github/workflows/ci.yml | grep -F 'branches: [main]'` exits 0.
- `grep -A2 '^  pull_request:' .github/workflows/ci.yml | grep -F 'branches: [main]'` exits 0.

**Done condition:** All three checks succeed.

### Task 2: CI-02 — Align `mix ci.all` with `verify-test` compile step

**What to do**

1. Open `mix.exs` → `aliases/0`.
2. Change the `"ci.all"` entry from `["verify.format", "verify.credo", "verify.test"]` to `["verify.format", "verify.credo", "compile --warnings-as-errors", "verify.test"]` so local runs match the `verify-test` job sequence after `mix deps.get` (developer already has deps).
3. Do **not** change `elixir-version`, `otp-version`, or Postgres image tags in `ci.yml` unless a separate failure proves runner breakage (`06-CONTEXT` milestone rule).

**Read first**

- `mix.exs`
- `.github/workflows/ci.yml` (verify-test steps order)
- `.planning/phases/06-ci-on-github/06-RESEARCH.md` (Mix parity table)

**Acceptance criteria**

- `grep -F '"ci.all": ["verify.format", "verify.credo", "compile --warnings-as-errors", "verify.test"]' mix.exs` exits 0.

**Done condition:** Grep succeeds and `MIX_ENV=test mix ci.all` exits 0 from a clean dev DB (Postgres available per `config/test.exs`).
