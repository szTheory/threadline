## Goal

Implement `Threadline.Semantics.AuditAction` Ecto schema matching the Phase 2 table, add `has_many :transactions` / `belongs_to :action` wiring on `Threadline.Capture.AuditTransaction`, implement `Threadline.record_action/2` in `lib/threadline.ex` (or dedicated module re-exported from root) per D-06: required `repo:`, optional fields per SEM-02, `{:error, :invalid_actor_ref}` (or closed atom set) before changeset for SEM-05, `{:ok, %AuditAction{}}` or `{:error, changeset}` otherwise. Add focused ExUnit coverage for SEM and ACTR-03.

## Tasks

### Task 1: `Threadline.Semantics.AuditAction` schema

**What to do:**

1. Add `lib/threadline/semantics/audit_action.ex` with `schema "audit_actions"` and all columns from migration (including `inserted_at` only).
2. `has_many :transactions, Threadline.Capture.AuditTransaction, foreign_key: :action_id`.
3. Update `lib/threadline/capture/audit_transaction.ex`: replace `field(:action_id, :binary_id)` with `belongs_to :action, Threadline.Semantics.AuditAction, foreign_key: :action_id, define_field: false` only if field already exists — prefer `belongs_to :action, ...` and let Ecto define `action_id` (remove duplicate field line).

**Acceptance criteria:**

- `mix compile --warnings-as-errors` exits 0.
- `grep -q "has_many :transactions" lib/threadline/semantics/audit_action.ex` succeeds.
- `grep -q "belongs_to :action" lib/threadline/capture/audit_transaction.ex` succeeds.

**Done condition:** Associations match SEM-03.

### Task 3: `Threadline.record_action/2`

**What to do:**

1. Public API: `Threadline.record_action(name, opts \\ [])` with **required** `repo:` in opts (D-06).
2. Support opts keys: `actor` or `actor_ref`, `status`, `verb`, `category`, `reason`, `comment`, `correlation_id`, `request_id`, `job_id`.
3. Normalize `name` to string before cast.
4. Validate actor via `ActorRef.new/1` | `new/2` — on error return `{:error, :invalid_actor_ref}` (or documented closed set) **without** building a changeset.
5. Return `{:ok, struct}` or `{:error, %Ecto.Changeset{}}`.

**Acceptance criteria:**

- `grep -q "def record_action" lib/threadline.ex` succeeds (or `lib/threadline/record_action.ex` if split, but then `Threadline` must delegate — grep `record_action` under `lib/threadline/`).
- ExUnit: anonymous actor persists (ACTR-03); invalid non-anonymous without id returns error tuple (SEM-05); happy path inserts row with `name` stored as text containing `"login"` (example).

**Done condition:** SEM-01, SEM-05, ACTR-03 covered in tests.
