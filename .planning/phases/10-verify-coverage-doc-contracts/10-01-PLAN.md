## Goal

Deliver **TOOL-01**: a **`mix threadline.verify_coverage`** task that composes **`Threadline.Health.trigger_coverage/1`**, reads **host-configured expected audited table names** (strings), fails closed when the list is **missing or empty** on verify runs, prints a **deterministic** human-readable table + one-line summary to **stdout**, exits **status 1** when any expected table is `:uncovered`, and is covered by **unit** tests (pure policy) plus **integration** proof of non-zero exit when an expected table lacks a Threadline trigger.

1. Add `lib/threadline/verify/coverage_policy.ex` with `@moduledoc` describing intersection semantics.
2. Implement `violations/2` — first arg: list of `{:covered | :uncovered, String.t()}` from `trigger_coverage/1`; second arg: `expected_tables :: [String.t()]` (unique names, **public** table names only).
3. Return **sorted** list of `{:uncovered, table}` for every `table` in `expected_tables` that appears as `{:uncovered, table}` in the first list; **also** return violation `{:missing, table}` (or equivalent tagged tuple) for each `table` in `expected_tables` that **does not appear at all** in the trigger_coverage result (should not happen for `public` tables if Health lists all user tables — still handle for robustness).

### Task 3: `mix threadline.verify_coverage` Mix task

**What to do**

1. Add `lib/mix/tasks/threadline.verify_coverage.ex` with `@shortdoc` **exactly**: `Checks configured audited tables have Threadline capture triggers (uses Health.trigger_coverage/1)`.
2. `run/1`: parse argv with `OptionParser` for optional `--no-color` (boolean, default false) only if you implement ANSI stripping; **minimum** is no flags.
3. Call `Mix.Task.run("app.config", [])` then ensure app started (`Application.ensure_all_started(:threadline)` or full app — match how other tasks in repo load repo).
4. Resolve repo: `Application.get_env(:threadline, :ecto_repos)` → first repo module; if empty, **`Mix.raise`** message containing substring **`ecto_repos`**.
5. Read `Application.get_env(:threadline, :verify_coverage)` as keyword; fetch `:expected_tables` key → list of strings. If **nil** or **[]**, **`Mix.raise`** message containing substring **`expected_tables`**.
6. Call `Threadline.Health.trigger_coverage(repo: repo)`; feed to `CoveragePolicy.violations/2`; print ASCII table (column headers **`TABLE`**, **`STATUS`**) and rows for **each expected table** with status **`covered`** or **`uncovered`** derived from tuples; then one stdout line containing **`summary:`** (case-insensitive substring `summary` acceptable) with counts.
7. If violations non-empty, `Mix.raise` is wrong for exit 1 — use **`System.at_exit(fn _ -> exit({:shutdown, 1}) end)`** before returning **or** `Mix.raise` with `:exit_status` not available — **concrete:** call **`exit({:shutdown, 1})`** after printing when violations exist (document in moduledoc). If no violations, **exit 0** normally.
8. **Do not** duplicate Health SQL — verify with ripgrep below.

**Read first**

- `lib/mix/tasks/threadline.install.ex`
- `lib/threadline/health.ex`
- `lib/threadline/verify/coverage_policy.ex`

**Acceptance criteria**

- `rg 'pg_tables|pg_trigger' lib/mix/tasks/threadline.verify_coverage.ex`; **exit code 1** (no matches).
- `grep -F 'Threadline.Health.trigger_coverage' lib/mix/tasks/threadline.verify_coverage.ex` exits **0**.
- `MIX_ENV=test mix compile --warnings-as-errors` exits **0**.

**Done condition:** All acceptance criteria pass.
