## Decision

**Path A: Carbonite** — Use `{:carbonite, "~> 0.16"}` as capture substrate.

## Rationale

Carbonite passes all four gate questions. It is actively maintained at v0.16.1 (released 2026-04-19), supports PostgreSQL 13+ (covering our ≥ 14 requirement), and critically uses a transaction-row mechanism — not `SET LOCAL`. The `carbonite_transactions` table has an `id` column of type `xid8` keyed by `pg_current_xact_id()`, and `insert_transaction/3` links metadata via a direct `INSERT ... ON CONFLICT DO NOTHING`. This is PgBouncer-safe and satisfies the schema constraint in D-06. The API is well-structured with a configurable `carbonite_prefix` option that allows Threadline to place audit tables in a non-public schema.

## API Surface Summary

**Schema installation:**
```elixir
Carbonite.Migrations.up(patch_or_range, opts \\ [])   # opts: carbonite_prefix
Carbonite.Migrations.down(patch_or_range, opts \\ [])
```

**Trigger management:**
```elixir
Carbonite.Migrations.create_trigger(table_name, opts \\ [])
# opts: table_prefix, carbonite_prefix, initially: :immediate | :deferred
Carbonite.Migrations.drop_trigger(table_name, opts \\ [])
Carbonite.Migrations.put_trigger_config(table_name, key, value, opts \\ [])
```

**Runtime (per-request/job):**
```elixir
Carbonite.insert_transaction(repo, params \\ %{}, opts \\ [])
# params may include :meta (map) for application-level metadata
```

**Schema prefix support:** YES — `carbonite_prefix` option on all migration functions.

**Features Threadline does NOT need in Phase 1:**
- `processed_at` / outbox marking pattern (Carbonite supports marking transactions as "processed" for event streaming use cases — Threadline does not use this)

**Conflicts with D-05 schema:**
- Carbonite uses `xid8` (PostgreSQL native type) as the transaction PK, not UUID or integer. Threadline's `AuditTransaction` will wrap this; no conflict, but Plan 01-02 must account for the `xid8` type when defining Ecto schemas.
- Column naming: `inserted_at` (matches Ecto convention). No conflicts found.

## Plan 01-02 Implications

- Add `{:carbonite, "~> 0.16"}` to `mix.exs` deps.
- Use `Carbonite.Migrations.up(1..12)` in the capture migration (or the current max patch level from hex.pm docs).
- Use `carbonite_prefix: "audit"` (or `"threadline_audit"`) to isolate audit tables from the `public` schema.
- `AuditTransaction` Ecto schema must map the `id` field as `Ecto.Type` `:integer` (xid8 maps to 64-bit integer in Postgrex) or use a raw type — confirm in Plan 01-02 spike.
- Call `Carbonite.insert_transaction(repo, %{meta: context_map})` at the start of each audited operation to attach `AuditContext` metadata.
- Do not implement `processed_at` tracking in Phase 1.
