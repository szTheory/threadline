## Tasks

### Task 1: `Mix.Tasks.Threadline.Export`

**What to do**

1. Create `lib/mix/tasks/threadline.export.ex` (`Mix.Tasks.Threadline.Export`) with `@shortdoc` one line.
2. **Strict flags** (extend as needed, document in `@moduledoc`):
   - `--output` / `-o` — required for non-dry-run write path (or allow `-` for stdout with explicit warning in moduledoc).
   - `--format` — `csv` | `json` (required unless defaulting to `json`—document).
   - `--json-format` — `wrapped` | `ndjson` (maps to `Threadline.Export` opt `json_format:`).
   - `--max-rows`, `--dry-run` — when `--dry-run`, call `Threadline.Export.count_matching/2` from 14-01 and print `%{count: n}` (no file write); no `max_rows: 0` hack.
   - `--table`, `--from`, `--to` — ISO-8601 strings parsed to `DateTime` (document UTC expectation).
   - `--actor-json` — optional path to a UTF-8 file whose contents `Jason.decode!/1` yields a map acceptable to `Threadline.Semantics.ActorRef.from_map/1` (`{:ok, ref}` → pass as `:actor_ref` in filters); on error, `Mix.raise` with clear message. If omitted, actor filter is unset (same as timeline).
3. Build `filters` keyword + `opts` and call `Threadline.Export.to_csv_iodata/2` or `to_json_document/2` only.
4. Write `:data` to file via `File.write!/2` or stream for large NDJSON (implementer choice; document memory caveat for huge NDJSON without streaming write).
