# Plan 23-01: HTTP audited path (`Threadline.Plug` + `POST /api/posts` + ConnCase proof)

**Phase:** 23 ‚Äî HTTP audited path  
**Plan:** 23-01 of 01  
**Status:** Ready to execute  

---

## Threat model

| ID | Threat | Severity | Mitigation in this plan |
|----|--------|----------|-------------------------|
| T-23-01 | `Threadline.Plug` or controller sets session-level GUC / runs `SET` on pooled connection | high | **Only** `Repo.query!(‚Ä¶, true)` inside **`Repo.transaction/1`** in **`ThreadlinePhoenix.Blog`** ‚Äî never in Plug or Endpoint (**D-08**, `Threadline.Plug` moduledoc). |
| T-23-02 | Mass assignment or unvalidated JSON writes attacker-controlled columns | medium | Use **`Post.changeset/2`** with **`cast`** limited to **`:title`**, **`:slug`** only; controller passes only **`params["post"]`** keys for those fields. |
| T-23-03 | README or tests imply fake API keys are auth | low | Synthetic **`ActorRef`** only (**D-04**); README states production replaces **`actor_fn`** (**D-05**). |

---

## Objective

Wire **`Threadline.Plug`** on the example **`:api`** pipeline, add **`POST /api/posts`** that creates a post through **`ThreadlinePhoenix.Blog`** ( **`Repo.transaction`** + **`set_config('threadline.actor_ref', ‚Ä¶, true)`** + audited **`Repo.insert`** ), expose a skinny **`PostController`**, and add a **ConnCase** test through **`ThreadlinePhoenixWeb.Endpoint`** asserting **`audit_changes`** for **`posts`** and **FK linkage** to **`audit_transactions`** ‚Äî satisfying **REF-03** and **23-CONTEXT** D-01‚ÄìD-11.

## must_haves

- [ ] **`router.ex`** includes **`plug Threadline.Plug`** with **`actor_fn:`** for a stable synthetic actor.
- [ ] **`POST /api/posts`** returns **201** with JSON for **`title`** / **`slug`** on valid body.

- [ ] ConnCase test hits **`/api/posts`** via **`post/3`** and asserts **`AuditChange`** for **`posts`** linked to **`AuditTransaction`** with **non-nil** **`actor_ref`**.
