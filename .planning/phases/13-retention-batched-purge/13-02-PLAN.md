## Tasks

### Task 1: Implement `Threadline.Retention.purge/1`

**What to do**

1. Public function `purge(opts)` — **require** `repo:` (`Threadline.Test.Repo` in tests).
2. Resolve cutoff via `Threadline.Retention.Policy.cutoff_utc_datetime_usec!/0` or pass `cutoff:` override **only if** documented as “stricter than policy” (tighter window) for ops emergencies — if not in scope, omit override.
3. Delete loop:
   - Target rows: `AuditChange` where `captured_at < ^cutoff` (exclusive upper bound on old data — match docs from 13-01; if docs say inclusive boundary, use `<=` consistently).
   - Batch: subquery `select id, … limit batch_size` then delete by ids (pattern from `13-RESEARCH.md` if `delete_all` + `limit` unsupported).
4. After each change batch (or end of iteration): run orphan **parent** delete for default `delete_empty_transactions: true` (read option from `:retention` config, default true).
5. Stop when zero rows deleted or `max_batches` reached.
6. Optional: `:telemetry.execute` event `[:threadline, :retention, :purge, :batch]` — if omitted, at least `Logger.info` with structured metadata `%{deleted_changes: n, batch: i}`.

**Read first**

- `lib/threadline/capture/audit_change.ex`, `lib/threadline/capture/audit_transaction.ex`
- `lib/threadline/retention/policy.ex`
- `13-CONTEXT.md` D-10, D-08–D-09

**Acceptance criteria**

- `grep -q "def purge" lib/threadline/retention.ex` exits 0.
- `MIX_ENV=test mix compile --warnings-as-errors` exits 0.
- `purge/1` without `repo:` raises `ArgumentError` or `KeyError` containing `repo` (document which; test asserts).

**Done condition:** API compiles; invalid opts fail fast.
