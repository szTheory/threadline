## Threat model

| ID | Threat | Severity | Mitigation in this plan |
|----|--------|----------|-------------------------|
| T-13-01-01 | Negative or zero retention window deletes everything or fails open | high | `Policy.validate!/1` (or equivalent) rejects non-positive durations; unit test `assert_raise` with message containing `retention` or `positive`. |
| T-13-01-02 | Ambiguous cutoff semantics vs `Query.timeline/2` | medium | `guides/domain-reference.md` documents that purge uses `captured_at` and states inclusive/exclusive boundary relative to `:from`/`:to` docs in `query.ex`. |
| T-13-01-03 | Undocumented extension hooks invite unsafe overrides | low | `@moduledoc` on policy module states v1.3 is global window only; per-table deferred per `13-CONTEXT.md` D-05–D-06. |

---

## Goal

Ship **RETN-01**: a single, documented **global retention window** resolved from application config (`13-CONTEXT.md` D-04, D-07), with a small **validation module** so bad config fails at boot or first purge resolution — not mid-delete. Document **what “expired” means** (`captured_at` vs `occurred_at`, row-level clock, long-txn edge case per D-01–D-02).
