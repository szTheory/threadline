<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → public API | `schema_module`, `id`, `timestamp`, and `opts` are untrusted inputs crossing into query construction |
| public API → database | the lookup must stay parameterized so history data is read without interpolation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-38-01 | T | `Threadline.Query.as_of/4` | mitigate | Use Ecto parameter binding for `table_name`, `table_pk`, and `captured_at`; derive table/PK metadata from `__schema__/1` instead of caller-provided fragments. |
| T-38-02 | I | reconstructed historical rows | mitigate | Return only the snapshot owned by the caller's explicit repo boundary; do not join unrelated tables or pull live-table data that could widen historical exposure. |
| T-38-03 | D | point-in-time lookup path | mitigate | Keep reconstruction to at most two indexed queries per call (latest snapshot, optional earliest snapshot probe) and reuse the same stable ordering as existing timeline reads. |
</threat_model>
