<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| audit_changes JSON -> struct loader | Persisted historical data crosses into typed Ecto loading. |
| public `cast: true` option -> query implementation | Callers can opt into struct reification, which must not widen the API beyond the locked behavior. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-39-01 | Tampering | `Threadline.Query.as_of/4` loading branch | mitigate | Use Ecto-native loading only; ignore unknown keys and return an explicit cast error instead of raw-map fallback when loading fails. |
| T-39-02 | Denial of service | historical snapshot loading | mitigate | Keep failure contained to a returned tuple and cover it with regression tests so malformed audit rows do not crash callers. |
| T-39-03 | Information disclosure | historical snapshot payload | accept | No new data source is introduced; the loader only projects the stored snapshot into the requested schema. |
</threat_model>
